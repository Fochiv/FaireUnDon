F. SUIVI EN TEMPS RÉEL DU STATUT DE PAIEMENT (Comportement critique)
Après validation du formulaire de don et initiation du paiement via l'API :
- Afficher un compteur dégressif de 8 minutes (08:00 → 00:00) indiquant à l'utilisateur qu'il dispose de ce délai pour valider sa transaction sur son téléphone.
- Simultanément, lancer une boucle de polling automatique qui interroge le statut de la transaction toutes les 3 secondes via le webhook/endpoint de l'API de paiement.
- Dès que le statut retourné passe à "validé" :
  * Mettre à jour le statut dans la base de données (côté admin ET côté utilisateur)
  * Arrêter le polling
  * Rediriger automatiquement vers la PAGE DE REMERCIEMENT
- Si le statut passe à "échoué" ou si le délai de 8 minutes expire sans confirmation :
  * Afficher un message d'erreur clair
  * Mettre le statut à "échoué" dans la base de données
  * Permettre à l'utilisateur de réessayer
- Ce mécanisme doit fonctionner entièrement de manière automatique, sans action manuelle de l'utilisateur.
